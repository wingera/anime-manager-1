# AGENTS.md

本文件约束当前项目的编码助手、人工提交和自动生成代码。所有代码、脚本、界面、文档、测试必须先满足本文件，再满足具体需求。

## 项目定位

项目名称：番剧自动整理管家。

项目目标：接入用户有权使用的资源来源，识别 40 位资源指纹，生成下载任务，匹配影视资料，联动下载器，分析多文件内容，把文件整理成媒体库可识别结构，并通过网页完成全部设置、预览和人工确认。

资源指纹 = 40 位十六进制字符串。
磁力入口 = 下载器能识别的下载入口。
影视资料 = 剧名、年份、季数、集数、集名、海报、简介、资料编号。
匹配分 = 系统判断“下载文件对应哪一季哪一集”的可信程度。
入库 = 把文件移动、复制或硬链接到媒体库目录，并记录旧路径、新路径和回滚信息。

## 技术栈

后端：Python 3.13.x、FastAPI、Pydantic v2、SQLAlchemy 2.x、SQLite、Alembic、APScheduler、httpx、pytest、ruff、mypy。

前端：Node.js 24 长期维护版、Vue 3、TypeScript、Vite、Pinia、Vue Router、Element Plus、vue-tsc、ESLint、Prettier。

外部能力：ffprobe 用于视频信息分析，qBittorrent Web API 用于下载器对接，TMDB API 用于影视资料查询，Docker Compose 用于本地部署。

默认数据库：SQLite，文件位置为 `data/app.db`。

进阶部署可选：PostgreSQL、Redis、独立任务队列。默认版本不得依赖这些组件。

## 命名规则

目录名和文件名使用行业通用英文专业名词，例如 `backend/`、`frontend/`、`docs/`、`scripts/`、`samples/`、`app/main.py`、`config/settings.py`、`db/models.py`、`migrations/`。

代码标识符使用稳定英文词表，禁止拼音缩写，禁止中英混杂。技术层沿用行业通用命名，例如 `router`、`service`、`client`、`schema`、`session`。业务词必须在 `docs/glossary.md` 里有中文解释。

用户可见内容必须是中文，包括网页、按钮、提示、日志、错误信息、接口返回给前端的提示、文档和安装说明。

第三方接口固定字段、框架关键字、库名按官方原样保留，但必须限制在对接层，不能把第三方原始结构直接泄漏到业务层。

## 后端代码规范

接口层只做参数校验、权限检查、调用业务服务、返回结果，不写下载、匹配、改名等业务细节。

业务层负责来源识别、资料匹配、下载任务、文件分析、命名预览、入库执行。

数据层只负责数据库模型、查询、事务，不调用外部接口。

对接层只放第三方接口封装，例如 TMDB、qBittorrent、ffprobe。第三方原始字段必须转换成项目内部模型。

公共函数必须写类型标注。接口入参和出参必须使用 Pydantic 模型。

数据库会话必须按请求或任务独立创建，不得跨线程、跨协程共享同一个会话。

日志统一通过日志工具输出，不使用 `print`。日志内容必须包含任务编号、阶段、动作、结果、失败原因。

异常必须转换成中文错误码和中文提示。内部异常细节只进入日志，不直接显示给普通用户。

文件操作必须先生成预览，再执行。执行时必须记录旧路径、新路径、操作方式、时间、结果。

路径处理必须使用 `pathlib.Path`，不得手工拼接 `/` 或 `\`。

外部请求统一使用 `httpx`，必须设置超时、重试次数和失败日志。

## 前端代码规范

页面文案必须中文。按钮、表单、空状态、错误提示、确认弹窗、日志展示都不得出现未解释的英文。

UI 参考 ANI-RSS 的信息组织方式：任务看板、来源管理、匹配中心、下载队列、文件分析、命名预览、入库记录、系统设置、运行日志。只参考交互思路，不复制源码、素材或默认来源。

组合式接口统一使用 `<script setup lang="ts">`。

状态管理统一使用 Pinia。跨页面状态放仓库，页面内临时状态放组件。

接口请求统一放在 `src/api/`，页面不得直接拼接后端地址。

类型定义统一放在 `src/types/`，不得在多个页面重复定义同一结构。

危险操作必须二次确认：删除来源、删除下载任务、执行入库、回滚入库、清空日志、修改媒体库目录。

所有设置项都必须有说明文字、默认值、测试按钮或保存后验证入口。

## 数据库规范

表名和字段名使用稳定英文词表，采用小写蛇形命名。

主键统一为 `id`。创建时间字段统一为 `created_at`。更新时间字段统一为 `updated_at`。软删除字段统一为 `deleted_at`。

资源指纹字段统一为 `resource_hash`，必须建立唯一索引。资源指纹保存前统一转为小写。

任务状态字段统一为 `status`，状态值固定为：`pending_review`、`pending_download`、`downloading`、`pending_analysis`、`pending_import`、`completed`、`failed`。

数据库迁移必须通过 Alembic。任何表结构变更必须同时提交迁移脚本和回滚说明。

## 任务状态规范

任务必须可重复执行，重复执行不得造成重复下载、重复入库、重复改名。

状态流转固定为：

```text
发现资源 → 待确认 → 待下载 → 下载中 → 待分析 → 待入库 → 已完成
                         ↓          ↓         ↓
                       已失败      已失败    已失败
```

失败任务必须保留失败阶段、失败原因、可重试次数、最后一次错误。人工确认后可以回到上一安全状态。

定时任务由 APScheduler 负责触发，真正的业务状态必须记录在数据库，服务重启后可以恢复。

## 来源识别规范

来源只能作为用户在网页中添加的授权来源。系统不得硬编码具体站点为默认来源。

来源配置必须包含：名称、类型、地址、启用状态、抓取间隔、授权备注、资源指纹规则、标题清洗规则、最后抓取时间。

资源指纹必须满足：长度 40，只包含 0-9、a-f、A-F。保存前统一转为小写。

同一资源指纹只能创建一个资源记录。重复发现时只更新来源记录和发现时间，不重复创建下载任务。

来源抓取失败不得影响其他来源。失败次数连续达到阈值后自动暂停，并在网页提示用户处理。

## 影视资料匹配规范

默认只匹配电视剧资料。电影、音乐、图片、游戏不得进入电视剧自动入库流程。

匹配流程固定为：标题清洗 → 候选搜索 → 详情补全 → 季集推断 → 计算匹配分 → 自动处理或人工确认。

匹配分满分 100：剧名相似度 40，年份相近 15，季集编号 30，文件数量与集数跨度 10，语言地区 5。

85 分及以上可以自动进入下一步。60 到 84 分必须进入人工确认。60 分以下只记录，不下载、不入库。

跨季合集、多剧合集、缺集、特典、无集数标题、文件名和资料库冲突时，必须进入人工确认。

人工确认结果可以保存为规则，但规则必须能在网页查看、禁用、删除。

## 下载器对接规范

下载器只通过 qBittorrent Web API 对接。

添加下载任务时默认先暂停，等待文件清单获取完成并完成多文件分析后，再按规则开始下载。

文件清单分析后，正片文件下载，样片、说明文件、明显无关文件默认不下载。低可信判断必须人工确认。

不得把下载器账号、密码、访问钥匙写入代码。网页设置保存时必须加密或使用系统密钥保护。

下载器连接失败必须显示中文原因：地址不可达、认证失败、权限不足、任务不存在、路径不可写、接口版本不支持。

## 文件分析规范

正片判断优先级：视频扩展名、文件大小、ffprobe 时长、标题集数、文件数量与资料库集数是否接近。

字幕判断优先级：扩展名、文件名是否与正片同名、语言标记、目录位置。

常见视频扩展名：mkv、mp4、avi、mov、wmv、flv、ts、m2ts。

常见字幕扩展名：ass、srt、ssa、vtt、sub、idx。

小于用户设置阈值的视频文件默认判为样片，但必须在网页可见并可手动改为正片。

ffprobe 失败时不得阻断整个任务，改用文件大小和文件名规则，并降低匹配分。

## 命名与入库规范

默认目录结构：`媒体库/剧名 (年份) [tmdbid=编号]/Season 01/剧名 - S01E01 - 集名.扩展名`。

特别篇目录：`Season 00`。

多集同一文件：`剧名 - S01E02-E03 - 集名.扩展名`。

入库方式必须支持移动、复制、硬链接。默认使用硬链接，硬链接失败时不得自动改用移动，必须进入确认。

同名冲突不覆盖，生成冲突任务等待确认。

## 测试规范

后端新增接口必须有 pytest 测试。核心匹配算法、资源指纹校验、文件分析、命名预览必须有单元测试。

前端新增页面必须通过 TypeScript 检查和构建检查。

提交前必须执行 `scripts/local_check.sh`。

## 禁止事项

不得内置未授权来源。

不得硬编码成人站点、盗版站点或高风险站点为默认来源。

不得编写绕过访问限制、绕过登录、绕过反爬、规避版权限制的代码。

不得自动下载低可信匹配结果。

不得在日志、接口返回、前端状态里明文显示密钥、账号、密码。

不得在没有预览和回滚记录的情况下执行改名、移动、复制或硬链接。
